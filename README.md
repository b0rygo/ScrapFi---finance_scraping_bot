ESP32 Market Sentinel: AI Stock Scanner & Telegram Bot

> **Current status:** Python / Raspberry Pi starter. The code in this repository is the Python (Raspberry Pi) implementation; the ESP32-S3 port described below is planned (commit `#1 starter without ESP`).

This project is a fully automated, ESP32-S3 microcontroller-based stock market monitoring system. It combines real-time data analysis, artificial intelligence (LLM) for market sentiment evaluation, and a private Telegram bot. The entire system operates continuously as an independent device on a home network (IoT).

Why was this project created? (Purpose)
Tracking the stock market "manually" is time-consuming and requires constantly staring at charts. The goal of this project was to create a physical, low-voltage "stock market assistant" that will do the heavy lifting for the investor.

Instead of paying for expensive information terminals, the system automatically searches for the best opportunities (Top Gainers), analyzes global news using artificial intelligence, and monitors for sudden price spikes (above a specified percentage) in 3-minute intervals, sending immediate alerts straight to the user's pocket (Telegram).

Main Features
    Morning AI Report (Daily Module): Every morning, the system fetches a list of the fastest-growing stocks on the US market. Then, it searches for the latest articles about them and sends them to an LLM model (e.g., OpenAI/Gemini) to generate a brief, natural language summary explaining why the stock is rising.
    Short-Term Scanner (3-Minute Interval): The script continuously monitors selected stocks. If a stock's price records a sudden jump by more than a defined threshold (e.g., 1.5% in 3 minutes), the system reacts immediately.
    Private Alerting (Telegram): Notifications are sent exclusively to a pre-programmed chat_id. No one else has access to the bot's data.
    Memory Optimization (Zero Leaks): The code strictly ensures RAM is freed after every JSON checking cycle, allowing the device to run uninterrupted for weeks.

Hardware Requirements
    Board: ESP32-S3 DevKitC-1 N16R8
    Flash Memory: 16 MB (for code and configuration)
    PSRAM: 8 MB (critical for handling heavy JSON files from financial APIs and SSL/HTTPS certificates for Telegram)
    Power Supply: USB-C cable connected to a stable 5V charger (min. 1A)

Technologies Used (Stack)
    Language: MicroPython / C++ (Arduino IDE)
    Stock Market API: Finnhub / Polygon.io / Yahoo Finance API (fetching quotes)
    News API: NewsAPI / DuckDuckGo
    AI / LLM: OpenAI API / Google Gemini API (sentiment analysis)
    Communication: Telegram Bot API (webhook / long polling over HTTPS)

Data Flow (How it works)
    Initialization: The ESP32 connects to the home Wi-Fi network and synchronizes the time (NTP).
    Scanner Loop: Every 180 seconds, the microcontroller sends an HTTPS GET request to the stock market server, fetching selected tickers.
    Filtration: The microcontroller parses the JSON response in PSRAM, discards unnecessary data, and compares the current price with the previous record.
    Trigger: If the percentage difference (Change %) > X, a text message is generated.
    Notification: The Wi-Fi module opens an encrypted connection to Telegram servers, and the phone app displays a notification. RAM is cleared for the next cycle.
